# NowPlace (나우플레이스)
장소 저장소, NowPlace

Website link: https://nowplace.vercel.app/

## 해커톤 작품요약서

### 팀명

차차차(Chachacha)

### 작품명

NowPlace(나우플레이스)

### 작품 주제

**자율과 창의 기반의 디지털 혁신의 촉진**: URL을 데이터베이스로 사용하여 현재 위치를 저장하는 지도 기반 서비스

### 작품의 설명(제안하는 해법, 이에 대한 근거)

- 기존의 지도 앱(NaverMap, KakaoMap)은 지도앱에서 제공하는 장소(e.g. 동춘당공원, 한남대학교, …)에서 사용자가 찾고 싶은 장소를 특정 장소의 이름을 입력하여 특정 장소를 저장할 수 있었습니다. 또한, 입력된 장소명 외로 도로명 주소, 지번 주소를 기반으로 사용자가 저장하고 싶은 특정 장소를 저장할 수는 있었습니다. 그러나 기존의 지도 앱이 이러한 장소를 저장할 수 있는 기능은 충분히 지원은 하였지만, UI가 너무 복잡했습니다. 그러므로 그저 “장소”만 저장하고 싶은 사용자의 욕구를 충분하게 충족 시키지 못했습니다.
- 그러므로 NowPlace는 “장소”만 저장하고 싶은 사용자의 욕구를 충분하게 충족시키도록 초점을 맞췄습니다. 기존에 지도 앱에서 제공하는 장소에 기반해서 장소를 저장하는 방식이 아닌, 지도 상에서 사용자가 저장하고 싶은 위치를 “더블 클릭”하여 장소를 저장할 수 있도록 했습니다. 그리고 기존의 지도 앱과는 다르게, 저장한 장소에 150자 내의 짧은 “매모”도 추가할 수 있게 하여 사용자가 이 장소를 무엇때문에 저장을 했는지, 아니면 이 장소에 어떤 추억이 있는지 등등과 같은 정보들을 함께 기입할 수 있게하여 사용자의 편의성을 높였습니다.
- 그리고 사용자가 그저 일반적인 장소들을 저장할 때는 문제가 크게 되지는 않지만, 친척의 집주소나 아니면 정말 나만 알고 싶은 그런 프라이빗한 장소들을 사용자가 저장을 할때, 이러한 정보들이 서버에 저장된다면 사용자가 보안상의 우려를 할 수 있어 NowPlace의 이용률이 저하될 수 있기 때문에, 기존의 지도 앱과는 다르게, 사용자가 남긴 정보들은 모두 “base64”의 형태로 변환하여 URL의 query string(?data=…)으로 저장하도록 했습니다.
- 더불어, URL에 장소들을 저장하다 보니 그저 사용자가 저장한 모든 장소를 저장하거나, 아니면 다른 사람들에게 공유하고 싶을때 그저 URL만 복사하여 메시지로 보내면 되니 장소 공유의 편의성도 높였습니다.

### 기존의 작품(해법)과의 차이점

- 원래의 지도앱은 사용자에게 친화적인 ui를 제공하지 않았지만, NowPlace는 사용자에게 보여지는 프론트엔드의 전체 부분을 사용자를 고려하여 사용자가 매우 편하게 서비스를 이용할 수 있도록 했습니다.
- 또, 장소를 저장할때 메모를 추가할 수 있도록 하여 사용자가 이 장소를 왜 저장했는지 등등에 대한 정보 또한 기입할 수 있도록 했습니다.
- 또, 장소를 단순히 서버 상에 저장하는 것이 아니라 URL 상에 저장하도록 하여 사용자가 안심하게 서비스를 사용할 수 있도록 했습니다.

### 결과

- 이 서비스가 많이 활성화되고 알려지게 된다면 많은 분들이 지금 저장하고 싶은 장소를 빠르고 쉽게 저장할 수 있을 거라 생각합니다.
- 그리고 기존에 길찾기, 네비게이션, 로드뷰, 위성 지도, 장소 리뷰, … 등등 부가적인 기능이 너무 많았던 지도 서비스가 아닌,  장소만을 저장하는 지도 서비스인지라 사용자들이 피로감 없이 서비스를 사용할 수 있을꺼라 생각합니다.

### 웹사이트 사용 방법

### 사용자가 제공하지 않는 페이지에 접속시

- 사용자가 제공하는 페이지(’/’) 외로 다른 페이지에 접속시 notfound 페이지를 보여주게 됩니다. notfound 페이지에는 “죄송합니다. 페이지를 사용할 수 없습니다.” 같은 메시지가 노출되며 NowPlace로 돌아가기라는 버튼을 제공하여 사용자가 홈페이지(’/’)로 돌아갈 수 있게 합니다.

### 사용자가 서비스 처음 접속시

- 사용자가 NowPlace에 처음 접속시 먼저 isDataLoading을 true로 설정하여 로딩 화면을 보여줍니다. 로딩 화면을 사용자에게 표시를 하긴 하지만, 로딩 화면 뒷 편으로는 KakaoMap 컴포넌트도 함께 불러옵니다. 이게 가능한 이유는 로딩 화면의 position을 absolute로 하고, z-index: 1로 설정하여 카카오맵 컴포넌트 위로 로딩 화면을 표시하게 됩니다.
- 그리고 geolocation api와 permissions api를 지원하는지 체크합니다. 지원한다면, 위치 권한을 사용자에게 받게 되며, 이러한 권한을 받게 되는 상태를 prompt라 하며 이러한 정보는 geoPermission에 저장됩니다. 만약 사용자가 위치 권한을 승인했다면 상태가 granted으로 바뀝니다. 그리고 isDataLoading을 false로 설정하여 로딩 화면을 없엡니다. 그리고 startWatchingPosition을 실행하여 사용자의 현재 위치를 가져오게 됩니다. 그리고 startWatchingPosition 함수는 NowPlace에서 나가기 전까지 실시간으로 사용자의 현재 위치를 가져오게 됩니다. 그리고 현재 위치를 다 불러왔다면 isCurPosFetched를 true로 설정합니다. 그리고 현재 위치는 또 curPos라는 변수에도 저장됩니다. centerPos는 카카오맵을 지도상에 표시하는 중심 좌표입니다.
- 그리고 현재 위치를 받은 후에는 handleBoundsChanged 함수를 실행하여 “현재 위치 주변의 데이터”를 마커의 형태로 불러오게 됩니다. 그리고 데이터는 URL의 data라는 name의 query string(https://nowplace.kr/?data=…)에서 불러오게 됩니다. 만약 data query string에 아무런 데이터가 없거나, 손상 되었다면 아무런 데이터를 불러오지 않습니다.
- 그리고 onCreate를 활용하여 카카오맵 컴포넌트가 다 불러왔을때를 감지합니다. 카카오맵 컴포넌트가 다 불러왔다면, mapRef라는 카카오맵 객체를 저장하는 변수에 kakao.maps.Map이라는 데이터를 저장하게 됩니다. 그리고 mapRef에 카카오맵 객체가 저장됬고, centerPos 또한 저장됬다면, updateAdminDongAddr 함수를 실행하여 지금 불러온 지도 상의 행정동 정보를 불러오게 됩니다. 그리고 행정동 정보는 curAdminDongAddr에 저장하게 됩니다.
- 만약 사용자가 위치 권한을 승인하지 않았다면, geoPermission의 상태는 denied가 됩니다. 그리고 permReqToggle을 true로 설정하여 위치 권한 요청 바텀 시트(위치 권한이 거부되어 승인해야 현재 위치를 사용할 수 있다는 일종의 도움말 바텀 시트)를 작동할 수 있게 합니다. 그리고 { lat: 37.575857, lng: 126.976805 } 라는 default 값을 curPos와 centerPos에 저장하게 됩니다. 그리고 행정동 주소나, 데이터도 이 default 값을 이용하여 불러오게 됩니다. 그리고 permReqToggle이 true이기 때문에 불러오는 과정에서 위치 권한 요청 바텀 시트를 사용자에게 보여주게 됩니다. 이 바텀 시트는 사용자가 축소할 수 있습니다. 그리고 사용자가 바텀 시트를 축소하면 setShowMoreToggle는 false가 됩니다.
- 그리고 이러한 과정이 끝나면 화면상에는 “더보기 버튼, 검색 버튼, 현재 위치 추적 버튼, 그리고 [현재 행정동 주소] 검색, 장소 추가, 장소 더보기, 현재 위치” 이라는 버튼이 화면상에 노출되게 됩니다.

### 현재 위치

- 현재 위치는 current svg의 형태로 마커로서 제공하게 됩니다. 현재 위치가 변경되면 이 마커 또한 위치가 바뀝니다. 그리고 이 마커를 클릭시 isTracking은 true가 되고 현재 위치 추적 기능에 돌입하게 됩니다. 이 현재 위치 추적 기능은 아래에서 설명하도록 하겠습니다.

### 더보기 버튼 클릭시

- 더보기 버튼은 사이트 로고와 copyright 표시로 제공합니다. 이 버튼을 클릭시 showMoreToggle이 true가 되고 더보기 바텀 시트가 화면 상에 표시됩니다. 그리고 사용자가 이를 축소하면 setShowMoreToggle은 false가 됩니다.
- 더보기 기능에서는 “서비스 설정이 있고, 서비스 설정은 위치 권한 상태, 모든 장소 저장하기, 모든 장소 삭제하기 기능”을 포함합니다. 그리고 이용 안내가 있고, 이용 안내는 “사용 방법, 문의하기 기능”을 포함합니다. 그리고 서비스 안내가 있고, 서비스 안내는 “서비스 이용 약관, 개인정보 처리 방침 기능”을 포함합니다.
- 위치 권한 상태 기능은 승인됨(granted), 거부됨(denied), 요청중(prompt) 이렇게 3개의 상태를 사용자에게 보여주게 됩니다.
- 모든 장소 저장하기 기능은 지금 현재 URL 전체를 클립보드에 복사하게 됩니다. 사용자는 클립보드에 저장된 URL을 나중에 저장을 해둔 후에 저장한 링크를 클릭하여 그때 저장했던 장소에 다시 접근해볼 수 있으며, 아니면 다른 사람에게 URL을 공유하여 그 받은 사람이 링크를 클릭하여 공유한 장소에 접근할 수 있습니다. 이 기능은 서버로 할 수 있는 것이 아니라 오로지 URL을 기반으로 동작하게 됩니다.
- 모든 장소 삭제하기 기능은 query string에서 data query string만 삭제하게 됩니다. 즉, http://localhost:3000/?data=JTdCJTIyZjZjODlmN… 이러한 데이터에서 http://localhost:3000/ 이렇게 ?data=… 이 부분만 삭제하게 됩니다. 전체 URL을 삭제하는 것이 아니므로 다른 쿼리스트링 부분은 건드리지 않습니다.
- 사용 방법, 문의하, 서비스 이용 약관, 개인정보 처리 방침 기능은 Notion 링크를 연결하여 제공하게 됩니다.

### 검색 버튼 클릭시

- 검색 버튼은 search svg의 형태로 제공하게 됩니다. 이 버튼을 클릭시 searchToggle이 true이 되고 검색 바텀 시트가 화면 상에 표시됩니다. 그리고 사용자가 이를 축소하면 searchToggle이 false가 되며, 뒤에 나올 변수들이지만, searchResult, searchValue가 undefined가 되며, searchVisible이 false가 됩니다.
- 검색 기능에서는 먼저 input box가 있으며, placeholder에는 “[처음에 불러온 행정동 이름] 검색”이 보여지게 됩니다. 그리고 사용자가 데이터를 입력하면 onChange로서 searchValue이 실시간 업데이트 되며, enter를 사용자가 입력시 handleSearchEnter가 실행됩니다. handleSearchEnter는 검색을 할 수 있게 하는 함수입니다. 만약 사용자가 데이터를 입력하고 enter를 입력하면 searchVisible이 true가 됩니다. 그러나 사용자가 아무런 데이터도 입력을 안하고 enter를 입력하면 searchVisible이 false가 됩니다. searchVisible은 검색 결과를 보여줄 건지 말건지를 결정하는 토글입니다. 그리고 사용자가 만약에 null이나 undefined한 값을 입력했거나, 아니면 공백 문자를 입력했다면, null을 searchResult에 저장하게 됩니다. 그리고 검색 결과를 보여주는 컴포넌트에서는 만약에 검색 결과가 null이면 “일치하는 결과가 없습니다”라고 화면상에 보여지게 됩니다. 만약 사용자가 이런 값들이 아닌 온전한 값을 입력했다면, Fuse라는 검색 엔진을 활용하여 저장한 장소를 검색하게 됩니다.
- 저장한 장소를 검색할때는 저장한 장소에 데이터 중에서 content, landLotAddress, roadNameAddress를 기반으로 검색하게 됩니다. 이러한 데이터들 중에서 하나라도 데이터가 겹치는게 존재하다면 searchResult에 저장한 장소 전체의 데이터를 배열의 형태로서 저장합니다. 그러나 데이터가 겹치는게 존재하지 않다면 null을 저장하게 됩니다.
- searchResult에 값이 저장되었다면 화면 상에 검색 결과를 보여주게 됩니다. 만약에 도로명 주소가 있다면 도로명 주소만 보여주며, 지번 주소만 있다면 지번 주소만 보여주며, 둘다 없다면 그저 사용자가 저장한 메모 데이터인 content만 보여주게 됩니다. 그리고 이러한 데이터를 클릭시 해당 장소로 focus가 됩니다(setCenterPos를 해당 장소의 position으로 설정합니다.).

### 현재 위치 추적 버튼 클릭시

- 현재 위치 추적 버튼은 tracking svg의 형태로 제공하게 됩니다. 그리고 그냥 처음 NowPlace를 실행했거나(즉, 처음 화면에 불러왔다면 이미 현재 위치 추적 기능을 활성화하는 것입니다. ), 만약에 이 버튼을 클릭햇다면 isTracking를 true로 설정하고 filled tracking svg로 바뀝니다. 그리고 지도에서 움직임을 발생시키면(zoom, 지도 이동) 다시 isTracking를 false로 설정하고 tracking svg로 바뀌게 됩니다. 그리고 만약에 isCurPosFetched가 false인데 geoPermission이 denied가 아니라면 이는 현재 위치를 불러오는 상태라고 규정하고, spin svg를 제공하여 현재 위치를 불러오는 듯한 svg를 제공하게 됩니다. 그리고 isCurPosFetched가 true가 되면 다시 filled tracking svg로 바뀌게 됩니다(처음 불러올때는 현재 위치 추적 기능이 활성화 되니까요.).
- 그리고 isTracking이 true면 centerPos도 계속 curPos의 값으로 변경하게 됩니다. 변경하므로서 카카오맵에서 보여지는 위치를 현재 위치를 중심으로 하여 보일 수 있도록 합니다.

### [현재 행정동 주소] 검색 클릭시

- [현재 행정동 주소] 검색 버튼은 말 그대로 [현재 행정동 주소] 검색의 형태로 제공하게 됩니다. 이 버튼은 사용자가 지도를 움직이면 활성화가 됩니다. 왜냐면, NowPlace는 지금 보여지는 지도 상에서만 데이터를 가져오기 때문입니다. 지금 보여지는 장소가 아니면 데이터를 불러오지 않으며, 실시간으로 데이터를 불러오지도 않습니다. 그리고 움직임을 감지하는 방법은 mapMovedToggle이 true가 되면 움직임을 감지했다고 판단합니다.
- [현재 행정동 주소] 검색 버튼을 클릭시 fetchDataToggle이 true가 되며 읽어들이는 중… 으로 버튼이 바뀝니다. 그리고 handleBoundsChanged() 을 실행하여 “지금 보여지는 부분의 장소들만” 데이터를 불러오게 됩니다. 그리고 다 불러와졌다면 fetchDataToggle를 false로 바꾸고, mapMovedToggle도 false로 바꾸어 [현재 행정동 주소] 검색 버튼을 감추게 됩니다.

### 장소 추가

- 장소 추가 기능은 지도를 더블 클릭하면 dataToAddToggle을 true로 설정하여 장소 추가 바텀 시트를 표시하여 화면 상에 표시하게 됩니다. 사용자가 축소를 하게 된다면 dataToAddToggle은 false가 되고 축소가 진행되게 됩니다. 그리고 더블 클릭시에 바로 지금 더블 클릭한 장소의 position(lat, lng)을 가져오게 됩니다. 이 position 정보는 dataToAddPos에 저장되게 됩니다.
- 장소 추가 바텀 시트에는 메모를 입력하세요라는 textarea를 제공합니다. 여기에 사용자가 메모를 입력시 contentData에 해당 메모가 저장됩니다. 그러나 사용자가 꼭 메모를 입력을 하지 않아도 상관 없습니다. 그리고 하단 footer button으로 장소 저장하기를 제공합니다. 장소 저장하기를 클릭시 saveData 함수를 실행합니다. 이 함수에 인자로 dataToAddPos와 contentData를 전달합니다.
- saveData 함수 실행시 먼저 도로명 주소와 지번 주소를 각각 lotAddr, roadAddr에 저장합니다. 만약 둘 다 데이터가 없다면 null, null을 값으로 저장하게 됩니다. 그리고 fetchDataFromUrl(fetchDataFromUrl이 가져온 데이터의 형태는 { id: { position: { lat: …, lng: … }, address: { landLotAddress: …, roadNameAddress: … }, content: … }, id: …. } 의 형태입니다.) 을 실행하여 url에서 데이터를 fetchedData라는 변수로 불러오게 됩니다. 만약 불러온 데이터가 없다면 빈 객체를 fetchedData에 저장하게 됩니다. 그리고 불러온 도로명 주소, 지번 주소, 장소를 추가할 position, 메모 데이터를 기반으로 data 변수에 값을 저장합니다. data는 이중 객체이며, 일단 { id: { … } } 의 형태로 저장합니다. id는 저장하는 장소에 독립적으로 부여되는 것으로 하이픈을 제외한 uuid로 하여금 부여하게 됩니다. 이러한 data 의 형태는 { id: { position: { lat: …, lng: … }, address: { landLotAddress: …, roadNameAddress: … }, content: … } } 의 형태입니다. 이러한 data와 fetchedData를 병합하여 mergedData에 저장한 후base64ArrayEncoder 함수에 전달하여 base64로 변환한 데이터를 dataToSave에 저장한 후 saveDataToUrl에 전달하게 됩니다. saveDataToUrl은 data query string에 직접 데이터를 추가하는 함수입니다. router.push(pathname + '?' + createQueryString(key, value)); 이러한 형태로 직접 데이터를 추가합니다.
- 그리고 장소를 정상적으로 추가를 완료했다면 다시 장소를 불러와서 데이터를 다시 지도 상에 보여주게 됩니다. 장소를 불러올때는 기존의 불러온 데이터는 유지하고, 새로운 데이터를 추가한 채로 화면에 표시하게 됩니다.

### 장소 더보기

- 장소 더보기 기능은 저장한 장소의 마커를 클릭시 보여지게 되는 바텀 시트입니다. selectedMarkerToggle가 true가 되면 장소 더보기 바텀 시트가 보여집니다. 그리고 더불어 장소 마커를 클릭시 selectedMarkerData 라는 변수에 장소 마커의 데이터가 저장됩니다.
- 장소 더보기 바텀 시트에는 사용자가 입력한 메모가 보여집니다.
- 그리고 도로명 주소도 보여집니다.
- 그리고 장소 공유하기 라는 기능을 제공합니다. 장소 공유하기는 지금 클릭한 장소의 데이터만을 저장한 url을 생성하여 사용자의 클립보드에 저장하게 됩니다. 이 기능은 sharePlace 함수로 동작합니다.
- 그리고 장소 삭제하기 라는 기능을 제공합니다. 장소 삭제하기는 지금 클릭한 장소의 데이터만을 data query string에서 제거하게 됩니다. 이 기능은 removePlace 함수로 동작합니다. removePlace 함수에는 데이터가 삭제되면 새로고침(다시 지도상에 데이터를 보여주는것)이 되는 코드도 있습니다.
- 또 메모 수정하기 기능도 제공합니다. 메모 수정하기 기능은 기존의 데이터에서 지금 클릭한 마커(장소 더보기를 한 마커)의 메모 데이터(selectedMarkerData.content)를 수정할 수 있는 기능입니다. 데이터를 수정하는 방식은, URL에서 데이터를 불러온 후, 지금 클릭한 마커의 데이터에서 content의 데이터를 변경한 후, 새롭게 변경한 데이터를 newData에 저장하고, 이 데이터를 base64로 변환한 후, saveDataToUrl 함수를 사용하여 URL에 반영하게 됩니다. 그리고 다시 지도의 데이터를 불러오게 됩니다. 이때 데이터를 불러올때는 기존에 불러온 다른 데이터는 유지한 채로 불러오게 됩니다.
- 메모 수정하기 기능의 전체 코드는 이와 같습니다.

```tsx
const fetchedData: any = fetchDataFromUrl('data');
const newData = {
  ...fetchedData,
  [selectedMarkerData.id]: {
    ...selectedMarkerData,
    content: editDataContent,
  },
};

const dataToSave = base64ArrayEncoder(newData);
saveDataToUrl('data', dataToSave);
/* 데이터 새로고침 */
const fetchedMarkers = transformNestedObjectToArray(newData);
const map: kakao.maps.Map = mapRef;
const bounds = map.getBounds();
const visible: any = fetchedMarkers.filter((marker: any) => {
  const position = new kakao.maps.LatLng(
    marker.position.lat,
    marker.position.lng,
  );
  return bounds.contain(position);
});
setVisibleMarkers([...visibleMarkers, ...visible]);
```

### 원래 작품과 차이점은?

- 원래의 지도앱은 사용자에게 친화적인 ui를 제공하지 않았지만, NowPlace는 사용자에게 보여지는 프론트엔드의 전체 부분을 사용자를 고려하여 사용자가 매우 편하게 서비스를 이용할 수 있도록 했습니다.
- 또, 장소를 저장할때 메모를 추가할 수 있도록 하여 사용자가 이 장소를 왜 저장했는지 등등에 대한 정보 또한 기입할 수 있도록 했습니다.
- 또, 장소를 단순히 서버 상에 저장하는 것이 아니라 URL 상에 저장하도록 하여 사용자가 안심하게 서비스를 사용할 수 있도록 했습니다.

### 결과

- 이 서비스가 많이 활성화되고 알려지게 된다면 많은 분들이 지금 저장하고 싶은 장소를 빠르고 쉽게 저장할 수 있을 거라 생각합니다.
- 그리고 기존에 길찾기, 네비게이션, 로드뷰, 위성 지도, 장소 리뷰, … 등등 부가적인 기능이 너무 많았던 지도 서비스가 아닌,  장소만을 저장하는 지도 서비스인지라 사용자들이 피로감 없이 서비스를 사용할 수 있을꺼라 생각합니다.

### 추후 개선사항

- 기존 지도앱이 지원하던 기능이었지만, 기존 지도앱의 스타일이 아닌 더욱더 사용자 편의에 초점을 두어서, 장소 검색 기능에 “키워드로 장소 검색하기” 기능을 추가하여 사용자가 저장하지 않은 장소를 검색할 수 있도록 하여 사용자의 편의를 증진시키고 싶습니다.
- 바텀 시트가 조금 불안정하게 나오는 그러한 사소하지만 사용자들이 느끼기에 조금 좋지않은 버그들을 해결하여 사용자들에게 더욱더 편안한 사용자 경험을 제공하고 싶습니다.
